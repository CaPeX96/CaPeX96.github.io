技术面试

  1. 项目实战方面：

    • Spring Boot

      • 源于Spring，基因良好

      • IOC：控制反转

        • DI：依赖注入，自动赋值
        • 为Spring容器添加组件的注解
        • @Configuration+@Bean
        • @ComponentScan+@Component or @Controller or @Service or @Repository
        • 为组件自动装配的注解
        • Spring规范@Autowired
        • Java规范@Resource（不支持优先注入）

        • Java规范@Inject（不支持required=false）

        • 组件注入：@Import

      • AOP：面向切面编程

        • 动态代理：在程序运行期间动态的将某段代码切入到指定方法指定位置进行运行编程的方式

        • 配置类田间注解@EnableAspectJAutoProxy开启AOP，拦截方法，正常执行一条链，异常一条链

      创建切面类

      ​ 类上的注解：@Component加入组件 @Aspect 切面类

      ​ 方法上的注解@Before @Around @After @AfterReturning @AfterThrowing

      Aspect切面 JoinPoint连接点 Pointcut切点 Advice增强 Target目标对象 Weaving织入

      业务逻辑和切面的交点是切点

      通过切点找到目标对象bean，增强对象，再织入会切面，业务逻辑执行增强后的bean

      • 声明事务：

      ​ @EnableTransactionManagement开启spring事务管理，

      ​ 给使用sql的方法加上@Transational注解（自动开启事务，提交，错误则回滚）

      ​ 在配置类中加入事务管理器bean，注入到IOC容器中

      • 简化编码：指引入一个spring-boot-starter-web的依赖就相当于引入了一系列依赖springmvc，内置tomcat，日志slf4j，junit测试，validate验证

      • 简化配置：大量默认配置

      • 简化监控：springboot-admin做监控（需额外加入依赖）

      • 简化部署：打包成jar，一键启动

      • 官推加民推，势不可挡

    注解驱动开发

    • Spring MVC(低耦合，高重用)

      • Mode：bean
      • View：页面jsp、html等
      • Control：Controller

    • 前端

  2. java部分：

    • 面向对象三大特性封装，继承，多态

      • 封装：典型例子JavaBean属性都private，用getter，setter操作

      • 继承：类与类产生关系，多态前提，继承extends 实现implements 单继承多实现

      • 多态：父类对象指向子类实例（左父右子）

    ​ 官方：定义规范

    ​ 程序员：面向接口编程

    ​ 厂商：提供满足规范的接口实现类

    • 集合类：

      • 数组
      • 优：随机访问效率高（根据数组下标查询）；搜索效率较高，可使用折半查找
      • 缺：内存连续且固定，存储效率低；插入和删除效率低
      • 链表
      • 优：不要求内存连续，内存利用率高；插入和删除效率高（只需改变指针指向）
      • 缺：不支持随机访问；搜索效率低（需要遍历）
      • Hash表
      • 优：搜索效率高；插入和删除效率高
      • 缺：内存利用率低（基于数组）；存在散列冲突

    set–> HashSet（无序）、TreeSet（有序，用二叉排序树）

    list–>ArrayList、LinkedList、(Vector)

    map–> HashMap、LinkedHashMap（插入顺序）、TreeMap（key的自然排序，二叉排序树）、(HashTable（同一时 刻一个写线程）、SychronizedMap、ConcurrentHashMap（锁分段，允许多线程修改不同的段）)

    括号内代表线程安全

    • 字符串：String、StringBuilder （StringBuffer）

    • 高级特性

      • 多线程

      • 反射

    在运行状态，对于任意一个类都能够知道这个类的所有属性和方法并调用

    • IO流

    • 网络编程

    • 数据库编程

      • JDBC(Mysql,oracle)

        • 导入数据库连接包如mysql-connector-java-5.1.48-bin.jar
        • 注册驱动（DriverManager.getConnection）
        • 获取Connection，Statement，ResultSet
        • 关闭Connection，Statement，ResultSet

      • DataBase（C3P0,Druid）

        • 导入连接池的包
        • 使用配置文件做配置（驱动的注册的信息和连接池的配置信息）
        • 加载配置文件
        • 获取数据库连接池对象DataBase（db.getConnection）
        • 获取Connection，Statement，ResultSet

      • Spring JDBCTemplate

        • 导包
        • 创建模板对象（new JDBCTemplate(ds)）
        • 使用模板提供的方法完成CRUD操作

      • 持久化层框架（把内存的数据存到硬盘（数据库））

        • JPA（ORM对象关系映射规范pojo与数据库表建立映射关系 操作实体类就相当于操作数据库表 ）

        • Hibernate：全自动ORM框架，自动映射，操作对象即操作数据库表，多表查询麻烦

        • MyBatics （半自动ORM，需要编写大量SQL）

        • Spring data JPA（默认使用Hibernate）

        • 实体类

        @Entity：Hibernate映射对象，作用实体类上，必要，剩下会根据默认规则自动映射

        @Table：实体类与数据库表映射

        @Id：主键声明，作用属性上

        @Column：实体类属性与数据库字段映射

        • Dao类

        @Repository：作用做Dao类上，spring提供

        extends JpaRepository则自带CRUD等方法，泛型是实体类型和主键的类型

        spring-data-jpa还支持通过解析方法名创建查询即自动添加SQL语句

        • User findByName(String name)
        • User findByNameAndAge(String name,Integer age)

        也可以通过@Query注解创建查询使用JPQL语句，:#{}轮番登场，JPQL学习

        • @Query(“from User u where u.name=:name”)

        • User findUser(@Param String name);

        • MyBaticsPlus

    • 数据库
